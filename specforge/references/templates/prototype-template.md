# 原型图模板

## 是否需要原型

只要需求涉及页面、入口、交互、编辑态、弹窗、抽屉、配置、列表、图表或流程可视化，就需要原型图。

## 建议拆分

| 场景 | 需要展示 |
| --- | --- |
| 入口态 | 用户从哪里进入，入口按钮/菜单/行操作是什么 |
| 操作态 | 用户点击后看到什么，哪些区域发生变化 |
| 编辑/配置态 | 表单字段、必填项、校验、保存/取消 |
| 确认态 | 二次确认、影响范围、确认/取消 |
| 异常态 | 无权限、加载失败、校验失败、任务失败 |

## 布局要求

- 桌面大屏中原型图单行最多展示 2 张。
- 平板或中等宽度仍保持最多 2 张，不要为了填满宽度压缩原型。
- 页面宽度小于等于 800px 时单列展示。
- 单张原型应保留足够宽度承载 Tab、表格列名、表单字段和按钮文案。

## 细节要求

- Tab：展示选中态和至少 2 个相邻 Tab。
- 表格：展示关键列名、状态标签、操作列。
- 表单：展示字段名、必填标识、输入控件、主次按钮。
- 筛选区：展示关键词、类型、状态、时间等关键条件。
- 图表：展示标题、指标、筛选、联动/下钻/编辑入口。
- 审批：展示状态、聚合数量、一键处理、分条处理。

## 不合格原型

- 只有灰色矩形，没有业务字段。
- 只有文字说明，没有可视化结构。
- 只有最终态，没有入口或关键操作。
- 状态、按钮、字段和需求无关。
