# 方案质量门禁

生成 HTML 预览前必须完成以下检查。未命中项要么补齐，要么在“问题”或“风险评估”中说明。

## 上下文

- 是否识别项目。
- 是否读取 `📦 项目总览.md`。
- 是否读取对应 `project/*.md`。
- 是否判断需求类型并读取相关 playbook。
- 是否避免输出敏感凭据。

## 需求完整性

- 是否说明用户角色和使用场景。
- 是否说明核心对象和业务规则。
- 是否列出高风险待确认问题。
- 是否给出合理假设。

## 原型清晰度

- 涉及前端体验时是否展示一张或多张原型图。
- 原型是否有 Tab、表格列名、字段名、按钮、状态标签、处理动作等必要细节。
- 是否覆盖入口态、操作态、编辑/配置态、确认态或异常态中的关键场景。

## 研发可落地

- 前端事项是否覆盖页面、组件、状态、交互、异常。
- 后端事项是否覆盖接口、数据结构、权限、错误码、状态枚举。
- 运维事项是否覆盖部署、资源、日志、监控、告警、回滚。
- 前期准备是否覆盖账号权限、环境、样例数据、依赖服务。

## 交付可评审

- 风险是否包含影响和规避建议。
- 排期是否拆阶段并给出工作量。
- HTML 是否离线可打开。
- 没有空标题、空列表或“暂无”占位。
