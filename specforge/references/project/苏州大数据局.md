# 苏州大数据局

## 项目定位

苏州大数据局项目用于对接苏州大数据局，包含模型训练和管理后台。

## 业务范围

- 模型训练
- 管理后台
- beta/prod 环境部署
- 与 szaic 相关的客户化能力

## 技术栈

| 项 | 内容 |
| --- | --- |
| 前端 | Vue3 + Vite + ElementPlus + TypeScript |
| Node | >=18 |
| 包管理 | pnpm |
| 主要环境 | test/beta/prod |

## 部署说明

1. 构建镜像：通过 Docker 仓库构建 `mlops-szaic-admin` 或 `mlops-szaic-console`。
2. 上传镜像：通过内部跳板机和脚本拉取镜像，具体凭据见密码管理器/联系负责人。
3. 部署：beta 通过 fuxi-beta.szaic.com；prod 联系相关负责人。

## 账号与权限

- 账号和密码：见密码管理器/联系负责人。
- 生产环境操作需要确认权限和审批。

## 常见研发规则

- 涉及管理后台时，必须明确角色权限、数据范围和操作审计。
- 涉及模型训练时，必须明确训练任务状态、资源消耗、失败原因、日志和回滚/重试。
- 涉及部署时，必须补充 beta/prod 差异、镜像版本、发布步骤和负责人。

## 常见风险

- 客户化环境和内部标准环境存在差异。
- beta/prod 部署链路较长，容易出现镜像版本或负责人沟通问题。
- 模型训练任务依赖资源和日志，状态不完整会影响问题定位。

## 数字人生成方案时要特别关注

- 管理后台或模型训练模块归属
- 客户环境差异
- 部署步骤和运维负责人
- 任务状态、日志、监控和资源
- 生产发布风险和回滚方案
