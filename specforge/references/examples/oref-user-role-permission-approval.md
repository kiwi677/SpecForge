# 样例：OREF 用户角色权限与举报审批

## 原始需求

```text
oref 用户管理
1、用户角色：给用户分配角色
2、角色权限：给角色分配权限
3、审批：给模型｜数据集｜skill 提供举报入口，用户举报后平台收到审批信息，单个模型｜数据集｜skill可能有多条举报需要聚合展示，支持一键处理，也支持分条处理
```

## 项目匹配

- 项目：OREF
- 置信度：高
- 知识库：`project/OREF.md`

## 需求类型

- 后台管理类
- 权限审批类
- 前端交互类

## 应用 playbook

- `playbooks/admin-console.md`
- `playbooks/permission-approval.md`
- `playbooks/frontend-interaction.md`

## 原型拆分

- 用户列表：展示用户名称、当前角色、状态、操作。
- 角色权限：展示权限范围、权限树、保存/取消。
- 举报入口：展示举报原因、补充说明、提交。
- 审批聚合：展示举报对象、举报数量、处理状态、一键处理、分条处理。

## 方案重点

- 外部用户权限和内部管理权限不能混淆。
- 单对象多举报需要聚合展示，同时保留分条处理。
- 一键处理和分条处理都要有审计记录。
- 对外错误提示不能暴露内部细节。
